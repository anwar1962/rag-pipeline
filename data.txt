dbt (data build tool) is a transformation tool used by data engineers to transform raw data inside a data warehouse. In a typical dbt workflow, raw data is first loaded into a warehouse like Snowflake or BigQuery by an ingestion tool. dbt then runs SQL models that transform this raw data into clean, structured tables ready for analysis. dbt follows an ELT approach — Extract, Load, then Transform — which means transformation happens inside the warehouse rather than before loading. dbt models are organized into layers: staging models clean and rename raw sources, intermediate models join and enrich the data, and mart models produce the final tables used by analysts and dashboards. dbt also handles testing and documentation. Engineers write schema tests to check for nulls, uniqueness, and referential integrity, ensuring data quality at every step. dbt runs are typically orchestrated by tools like Airflow or Prefect, which trigger dbt on a schedule or when upstream data arrives. The result is a reliable, version-controlled, testable data transformation pipeline that the whole team can trust.
